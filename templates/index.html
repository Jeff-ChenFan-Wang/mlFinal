About Live Demo

Live BMI

Welcome to Jeff's Live BMI demo!

This project is meant to replicate the results of Face-to-BMI: Using Computer Vision to Infer Body Mass Index on Social Media (Kocabey et al. 2017).

 

Web App Architecture

Frontend webcam streaming is done locally on the client side via javascript, then sending snapshots at timed intervals to the server. This makes the app much more responsive and self contained, instead of needing to mess with STUN servers and having WebRTC bog down our backend (loading tensorflow models is already reaching 800MiB with just one worker, so I'm trying to shave off anything I can).

The Backend is supported by Flask and Gunicorn, and the webapp is deployed on a GCP Compute Engine instance via GAE.

 

Backend Model

The model uses Haar Frontal Face Cascade Classifier for multiple face detection through the webcam. The face detector uses the default 1.1 scaling, but with min-neighbors tuned down to 2 for more generous detection. The recognized faces are then extracted for BMI prediction. The bounding boxes are expanded slightly as the default bounding boxes were found to be too tight and was impacting the prediction slightly.

A Resnet-50 model trained on the VGG-Face (Parkhi et al. 2015) dataset for facial recognition is used to extract face embeddings from the faces recognized. The embeddings are then used for BMI prediction. The original paper then fed the embeddings through an epsilon support vector regression model to predict subject BMI. In our case, I wanted to put a personal spin for fun so I am using ridge regression instead for the final predictions.

Ridge performed much better than Lasso and Elasticnet. This is likely because a lot of features may be noisy and contribute little to the final accuracy, but completely zeroing most of them out erases a lot of information. Therefore, L2 regularization was much more desirable than L1 as it tapers noisy features towards zero but not completely to zero.

 

Test Results

As seen below, this approach achieved similar overall performance and outperformed the original approach for predicting the BMI of males very slightly, though sacrificing some performance on predicting female BMI.

Paper used Pearson r correlations on the test set for evaluation

Model Male Female Overall
VGG-Face + SVR 0.71 0.57 0.65
VGG-Face + L2 Regression 0.72 0.55 0.65

Both approaches outperform the trivial predictor by around 37% overall. The regression approach eeked over the original approach in the paper a tiny bit in overall MSE.

Model Male Female Overall

Disclaimer

This is mostly for fun to test model building and deployment capabilities. The author of the paper themselves said that any machine learning prediction method of individual characteristics using soley their face as a predictor may work “better than random guessing”, but would still only be useful for studying the general population and would be wildly inaccurate for any individual.
